# MFLI 维修项目对话调度与文件更新规则

本文档用于防止长对话压缩造成信息丢失。原则是：主对话负责决策，子对话负责长期专题工作，项目文件负责长期记忆。

## 1. 对话分工

### 主对话：MFLI维修-总控

职责：

- 维护当前项目状态和下一步优先级。
- 接收子对话结论并判断是否写入工程文件。
- 更新或审查 `docs/repair_status.md`、`README.md`、`tasks/*.md`、`measurements/*.csv`。
- 做高风险决策，例如隔离串联电阻、更换 AD9269、恢复输入保护器件。
- 检查 Git 状态、diff 和是否存在未审查的大文件。

主对话不应长期承载大量手册摘录、照片逐张分析或多轮测量细节。

### 子对话 A：MFLI维修-测量计划与记录-v2

职责：

- 把测量任务拆成可执行步骤。
- 维护上电/断电测量清单、测点、单位、仪器状态和安全注意。
- 整理 ORA/ORB、AVDD、DRVDD、VREF、VCM、RBIAS、A/B 钳位器件等测量结果。
- 输出可写入 `measurements/*.csv` 和 `tasks/*.md` 的内容。

边界：

- 不单独决定拆焊、隔离电阻或更换芯片。
- 不把未测量数值写成事实。

上下文管理：

- 旧线程 `019eea97-c639-72d2-8036-f7a815b7b9eb` 已因上下文窗口耗尽归档，只作为历史只读参考。
- 当前使用 v2 线程 `019eede0-0110-7c13-9485-7b865244dfb5`。
- 测量子对话必须采用文件优先方式，只读取当前任务必需文件，避免长历史堆积。

### 子对话 B：MFLI维修-电路分析与故障判断

职责：

- 基于测量结果和照片判断故障源头。
- 分析 AD9269 Pin61/Pin62、前级驱动、钳位网络、串联电阻隔离实验结果。
- 给出“前级故障 / ADC B 通道输入级故障 / 仍需补测”的判断依据。
- 输出可写入 `docs/repair_status.md` 和 `tasks/*.md` 的结论草案。

边界：

- 不凭记忆生成未经核实的芯片型号、参数或替代料号。
- 不直接推动高风险维修动作，必须回到主对话确认。

### 子对话 C：MFLI维修-Git与工程维护

职责：

- 维护本项目 Git/GitHub 工作流。
- 解释 `status`、`add`、`commit`、`remote`、`push` 等命令的作用、检查方法和回退方式。
- 提交前检查暂存范围、忽略规则、大文件和敏感资料风险。
- 给出适合 Windows PowerShell 的命令。
- 在主对话确认变更范围后，直接执行正常项目管理所需的 `git status`、`git diff`、`git add`、`git commit`、`git push`，用户无需手动操作 PowerShell。

边界：

- 未经主对话确认变更范围，不执行 `git add`、`git commit`、`git push`。
- 不执行 `git reset --hard`、`git clean -fd`、强制推送或改写历史。
- 不把视频、zip、密钥、凭据或未确认公开的维修资料上传到 GitHub。

## 2. 何时开新子对话

只在满足以下条件时才增加子对话：

- 任务会持续多轮，并且需要大量上下文。
- 任务结果会沉淀为项目文件。
- 放在主对话会明显干扰项目调度。

以下情况不新开子对话：

- 记录一个单点电压。
- 改一个文档小错误。
- 查一次 Git 状态。
- 生成一个简单 PowerShell 命令。

## 3. 子对话回传格式

每个子对话给主对话的结论统一使用：

```text
来源子对话：
任务目标：
输入资料：
关键结论：
证据位置：
不确定项：
建议修改文件：
建议下一步：
```

主对话收到后再决定是否写入项目文件。

## 4. 文件作为长期记忆

关键事实必须落到文件，不只保留在对话中。

| 内容 | 文件 |
|---|---|

| 项目调度规则 | `docs/project_coordination.md` |
| 待办任务 | `tasks/issue_*.md` |
| 测量数据 | `measurements/*.csv` |
| 原始照片证据 | `images/raw/` |
| 历史分析 | `docs/prior_analysis/` |
| Git 入口说明 | `README.md` |

## 5. 写入规则

- 测量数据写入 CSV 时必须包含日期、测点、单位、上电/断电状态和来源说明。
- 推断性结论必须标明依据；不能把假设写成已验证事实。
- 涉及拆焊、隔离、上电测量的步骤，必须保留安全注意。
- 修改后运行 `git status --short --ignored`，必要时检查 `git diff`。
- 未经用户要求，不执行 `git add`、`git commit`、`git push`。
